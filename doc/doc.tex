 % <<< Introduction
 \chapter{Introduction}
 The rapid development of low-cost microcontroller platforms has created new possibilities for the development of versatile robotic systems.
 The Arduino ecosystem has emerged as the most popular open-source prototyping platform because of its accessible hardware architecture, extensive library support and the large community around it.
 By integrating Bluetooth communication, it becomes possible to create a wireless link between a microcontroller-based vehicle and a smartphone, enabling real-time control via a dedicated mobile application.

 This paper presents the design, implementation and source code of a smartphone controlled robotic car built upon the Arduino platform.
 The system uses the \texttt{Serial Port Profile (SPP)} of the Bluetooth standard to transmit directional control signals from a mobile device to the Bluetooth module (\texttt{HC-05}), which gets parsed by the microcontroller to drive a motor driver module (\texttt{L298N}) that actuates the vehicle's DC motors.
 The modularity, simplicity and extensibility of the architecture allow it to serve as a foundational platform for a wide range of applications.
 These include, but are not limited to:
 \begin{itemize}
  \item remote-controlled (RC) navigation,
  \item autonomous maze-solving through wall-following algorithms,
  \item obstacle avoidance via ultrasonic or infrared proximity sensors,
  \item line-following using reflectance sensor arrays,
  \item robotic manipulation through the integration of a robotic arm.
 \end{itemize}

 The flexibility of the platform makes it well-suited for both educational environments and practical deployment scenarios.

 \section{Software and Tools}
 % <<< IDE
 \subsection{IDE}
 Software development was carried out using the Arduino build toolchain.
 While the Arduino IDE (\inserturl{https://docs.arduino.cc/software/ide}) provides an accessible interface for compilation and flashing, the project used Arduino-Makefile by Sudar Muthu (\inserturl{https://github.com/sudar/Arduino-Makefile}), allowing development in a custom environment (in this case, Neovim).
 % >>>

 % <<< Libraries used by the code
 \subsection{Libraries used by the code}
 To interface with the dual H-bridge motor driver, an external library was used: the open-source \texttt{L298N} library by Andrea Lombardo (\inserturl{https://github.com/AndreaLombardo/L298N}), specifically utilizing the \texttt{L298NX2.h} header, which provides a high-level abstraction for the simultaneous control of two independent motor channels.

 % <<< Configuring the mobile application
 \subsubsection{Configuring the mobile application}
 The mobile application requires the configuration of four directional control buttons, each mapped to a distinct single-character command transmitted over the Bluetooth serial link: \texttt{N}, \texttt{S}, \texttt{W}, and \texttt{E} corresponding to forward, reverse, left, and right motion respectively.
 An additional button is assigned the null command character \texttt{0}, which triggers a full motor halt.
